## Security

このドキュメントは `my-first-ai-app` の「最低限のセキュリティ方針」をまとめたものです。
（本番運用・外部公開をする場合は、WAF / Bot対策 / 監視・アラートなどを追加してください）

---

## Secrets（APIキー）

- Anthropic の API キーは **絶対にコミットしない**（`.env.local` / 環境変数で渡す）
- リポジトリには `.env.example` のみ置く
- CI では実APIを叩かない（E2Eはモックを使う）

---

## Rate limit / abuse 対策

`POST /api/chat` は簡易レート制限を持ちます。

- 目的: 外部API利用コストの暴発、連打によるDoSを抑制する
- 現状: **プロセス内 Map**（単一インスタンス前提）

本番で水平スケールする場合は、Redis 等の共有ストアに移して一貫した制限をかけてください。

---

## Error handling

API は主に以下のステータスで返します。

- `400`: 入力不正（`messages` がない等）
- `401`: APIキー未設定などの認証系
- `429`: レート制限超過
- `500`: それ以外のサーバーエラー

クライアント側は `response.ok` を確認し、`error` を表示します。

---

## Logging

本番運用では以下をログ/メトリクスで追うことを推奨します。

- `429` の発生回数（abuse兆候）
- `401` の発生回数（設定ミスの検知）
- `/api/chat` のレイテンシ・エラー率
